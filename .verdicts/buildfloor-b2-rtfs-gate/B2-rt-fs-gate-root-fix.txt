────────────────────────────────────────────────────────────────────────────
ROOT CAUSE (verified)
────────────────────────────────────────────────────────────────────────────
self/runtime.c had a 3-way gate around rt_fs_append_atomic / rt_fs_stat /
rt_fs_rotate_if_over:

  #if defined(HEXA_RT_SELFEMIT)              -> extern (self-emit .o supplies)
  #elif !defined(HEXA_HAS_HEXA_RT_STDLIB)    -> real failure-default C bodies
  #else  (== HEXA_HAS_HEXA_RT_STDLIB set)    -> extern, NO DEFINER  <<< BUG

tool/build_aprime.sh stage-4 prepends `#define HEXA_HAS_HEXA_RT_STDLIB 1`
(build_aprime.sh L119-120) and inlines runtime.c, so it hits the `#else`
branch. The builtin-init (runtime.c L13378-13380) unconditionally takes the
address `(void*)rt_fs_append_atomic` etc. via hexa_fn_new(), but the `#else`
externed the bodies away claiming "aprime_cc hexa transpile defines _rt_fs_*".

That claim is FALSE:
  - self/codegen.hexa:7205 / 7311 / 7376 only LOWER the bare `fs_stat` /
    `fs_append_atomic` / `fs_rotate_if_over` builtins to a CALL into rt_fs_*.
    codegen never emits a runtime function BODY (it emits user code + extern
    fwd-decls + a `#include "runtime.h/.c"` — never runtime impls).
  - self/stdlib/fs.hexa L223-224 documents these as runtime builtins
    ("fs_append_atomic is wired as a runtime builtin (codegen → rt_fs_*)").
  - No tracked .hexa / .c emitter defines _rt_fs_*.

=> Undefined symbols at clang link. #2421 worked around with a stage-3 .sh
   link-fill of 3 stubs identical to the !HEXA_HAS_HEXA_RT_STDLIB bodies.

Sibling rt_fs_mkdir_p (defined unconditionally right below) + rt_append_file
+ rt_write_bytes all carry UNCONDITIONAL real C bodies in runtime.c and
resolve under every config — that is the canonical convention.

────────────────────────────────────────────────────────────────────────────
FIX (chosen: option b — resolve in runtime SSOT, matching sibling convention)
────────────────────────────────────────────────────────────────────────────
Collapse the broken `#else` extern-away branch into the same failure-default
bodies, so HEXA_HAS_HEXA_RT_STDLIB links with NO link-fill stub. Only
HEXA_RT_SELFEMIT keeps the extern. Gate becomes:

  #if defined(HEXA_RT_SELFEMIT)   -> extern
  #else                            -> real bodies (covers BOTH the standalone
                                      smoke path AND the HEXA_HAS_HEXA_RT_STDLIB
                                      stage-4 path)

Net change: delete the `#elif !defined(HEXA_HAS_HEXA_RT_STDLIB)` line and the
dead `#else extern;` block; keep one body block under `#else`. (No deletion of
behavior — the bodies are identical to the pre-existing !STDLIB bodies.)

Option (a) (codegen.hexa emits the C bodies) REJECTED: codegen has no
runtime-body-emission mechanism; every builtin lowers to a CALL. Inventing
one contradicts the universal builtin→CALL+runtime.c-body architecture and is
far larger than the SSOT one-liner. Not the smallest correct change.
