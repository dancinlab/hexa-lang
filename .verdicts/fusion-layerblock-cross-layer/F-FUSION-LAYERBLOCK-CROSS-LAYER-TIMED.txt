ROOT CAUSE (read directly from block_fused_k1.ptx, lines 230-282):
  K1's flash-attention key-loop ($kloop) is an EMPTY STRUCTURAL STUB. The PTX
  comment is explicit — "structural pattern only this round", "omitted body
  (proven in flash_attn.ptx)". The loop body only does `add.s32 %r16,%r16,1`
  (increment kj); it computes NO dot product, NO online-softmax update, NO
  acc/l accumulation. Therefore at $kloop_done: acc_local[*] = 0 (init) and
  l (running sum %f43) = 0. Then:
    rcp.rn.f32 %f44, 0   -> +Inf
    normalize: acc[i] = 0 * (+Inf) -> NaN
    OUT-proj : fma(NaN, Wo, ..) -> NaN ; +residual1 -> NaN  -> mid = NaN
    K2 LN2(NaN) -> NaN -> block output = NaN  (all 3072 elements).

  K2 (block_fused_k2.ptx) IS numerically complete (full SwiGLU FFN: LN2 +
  up/gate proj + SiLU + down proj + residual2), but it is fed NaN from K1, so
  the whole-block output is NaN regardless.

  (An earlier harness build silently PASSED with max_rel=0 because `NaN > 0`
   is false in C — the NaN/Inf hard-fail counter `n_nan_inf` was added to
   close that false-pass; it reports 3072/3072 bad.)

RULED-OUT AXIS (closed-negative, the publishable finding)
--------------------------------------------------------------------------------
The round-8 axis-C STRUCTURAL moat (3 launches vs 11; 9.667× less activation-HBM
at dff=4d) is REAL and unchanged. But the structural moat DOES NOT CONVERT to a
timed above-library win with the as-merged kernels, for TWO independent reasons:

  (1) K1's attention seam is a stub (empty key-loop) — the fused kernel does
      not actually compute attention, so it cannot be numerically faithful.
  (2) K2's inner GEMMs are NAIVE SCALAR (one thread per row, no tiling) — a
      single launch is ~68 s, ~3 orders of magnitude over the 116 ms eager
      block. Even a perfect K1 could not rescue this.

This is EXACTLY the axis-B precedent the round-8 result.json pre-registered:
"structural != wall when inner GEMMs are scalar" (cf. F-FUSION-ATTN-WMMA 🔴,
F-FUSION-ATTENTION-FLASH 🔴 — naive scalar fused GEMM loses to a tiled lib GEMM).

To convert the axis-C structural moat to a timed win the fused block needs:
  - a real flash-attention inner body in K1 (not an empty stub), AND
  - Tensor-Core / tiled-shared-memory inner GEMMs in BOTH K1 (QKV/OUT) and K2
    (FFN up/gate/down) with multi-CTA occupancy (cf. F-FUSION-ATTN-MULTICTA).
