Source claims that this artifact synthesizes
--------------------------------------------

(W) WRITE-SIDE HAZARD — install-dir leak
    Mechanism:
      hexa atlas register --from-verify <fn> <n> <v>
      writes to install-dir embedded.gen.hexa (~/core/hexa-lang/compiler/atlas/embedded.gen.hexa)
      regardless of cwd. The install-dir git tree is the SHARED 8-session
      working tree; another agent's active branch HEAD turns the register
      output into a working-tree leak on that branch.
    Empirical witness:
      E1 PR #1070 (2026-05-25T12:07:46Z merged) — 6 verified-* nodes
      (tau-496, tau-8128, tau-33550336, is_perfect-8589869056,
       gamma0_genus-6, gamma0_cusps-6) folded; concurrent active branch on
      shared tree was `antimatter-h1s2s-rydberg-verify`; recovery needed.

(R) READ-SIDE HAZARD — binary-builtin freeze (complement)
    Mechanism:
      hexa atlas lookup reads binary-builtin tables baked at last hexa
      binary build. Source SSOT updates from register do NOT appear in
      lookup until the installed hexa binary is rebuilt.
    Empirical witness (from .verdicts/tecs-l-atlas-health/binary_vs_source_divergence.txt):
      binary-builtin total nodes         = 16101
      binary-builtin verified-* hits     = 74  (others, not E1 6 nodes)
      binary-builtin E1 6 nodes findable = 0    <-- gap
      source embedded.gen.hexa E1 6 nodes present = 6
      tier (E2)                          = SUPPORTED-BY-CITATION
                                           (out-of-band divergence)
      E2 PR #1096 merged 2026-05-25T13:08:05Z.

(W) and (R) are complementary: register updates source, query reads binary.
Together they explain why an axis-E fold can be present and absent
simultaneously, depending on the surface inspected.

Patch-to-worktree recovery (4 steps, used in E1 PR #1070)
---------------------------------------------------------

Step 1 — capture the leaked diff
    cd ~/core/hexa-lang
    git diff compiler/atlas/embedded.gen.hexa > /tmp/atlas-fold.patch
    # single-file SSOT => single-file patch is sufficient.

Step 2 — revert the shared tree immediately (protect other agents)
    git -C ~/core/hexa-lang checkout -- compiler/atlas/embedded.gen.hexa
    # the leak is removed from working tree; patch survives in /tmp.

Step 3 — make an isolated worktree on origin/main
    git -C ~/core/hexa-lang worktree add -b atlas-fold-<topic>-YYYY-MM-DD \
      /tmp/wt-atlas-fold origin/main
    cd /tmp/wt-atlas-fold

Step 4 — apply, validate, commit, PR
    git apply /tmp/atlas-fold.patch
    # (strip stray nodes / fix typos here if needed)
    git add compiler/atlas/embedded.gen.hexa
    git commit -m "feat(TECS-L): axis E ... fold N verified discoveries"
    git push -u origin atlas-fold-<topic>-YYYY-MM-DD
    gh pr create --base main --title ...

Concurrency note
    compiler/atlas/embedded.gen.hexa is a 16k+ line generated SSOT. Two
    parallel atlas-fold PRs touching nearby hunks will conflict (codegen-
    grade serialization recommended). On rebase conflict keep BOTH node
    sections (union — each node is an independent verified discovery).

Forward recommendations
-----------------------

(rec-1) Atlas write = single-writer serialization per round.
        Mirrors the codegen-PR pattern. Strongest preventative.

(rec-2) Query reads source SSOT preferentially. INBOX upstream
        (INBOX.log.md 2026-05-25T18:00Z) registers two options for the
        hexa-lang side fix:
          (i)  hexa atlas lookup should read HEXA_ATLAS_EMBED env or cwd
               compiler/atlas/embedded.gen.hexa BEFORE binary-builtin.
          (ii) OR hexa atlas register should mutate binary-builtin
               in-memory state after fold.
        Either closes the (R) hazard.

(rec-3) Periodic hexa binary rebuild after a batch of register folds
        (hexa cc --regen + install). Closes the source/binary gap on
        an N-PR cadence.

(rec-4) Pre-register self-check (one liner before each register):
        git -C ~/core/hexa-lang status --short | grep -v '^??' \
          || echo "OK: shared tree clean -> register safe"
        git -C ~/core/hexa-lang branch --show-current
        # if not 'main' or not clean, prepare Step 1-4 first.
