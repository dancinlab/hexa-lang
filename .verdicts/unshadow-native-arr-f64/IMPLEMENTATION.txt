  self/runtime_core_emit.hexa:

      — layout matches HexaArr/HexaArrI64 (ptr,int,int) so a TAG_ARRAY HexaVal
      carries it via the SAME arr_ptr union slot; polymorphic hexa_len reads len
      at the same offset.
    + helper bodies: malloc'd contiguous double[], realloc-grow push, len reader,
      and hexa_arr_f64_box (returns hexa_float(a->data[i]) at the dynamic boundary).

  self/codegen.hexa:
    + _is_float_literal_array(node): exact monomorphic-f64 proof (every element a
      FloatLit or UnaryOp(-, FloatLit)); float twin of _is_int_literal_array.
    + _known_floatarr_set + _known_floatarr_add/void/_is_known_floatarr_name:
      lifecycle twin of _known_intarr_*.
    + LetStmt arm: register `let xs = [f64-lit…]` as known-float-array; when the
      binding auto-fires (PART 2), lower to native contiguous double[]:
        ({ HexaVal __nfa = hexa_arr_f64_new(N);
           hexa_arr_f64_push(__nfa, (double)HX_FLOAT(<lit>)); ... __nfa; })
    + Index read arm: proven in-range read of a known-float-array, when native
      storage fired, emits the RAW native double load as an INLINE TAG_FLOAT

      re-box behind the wall and defeat the gather):

      else falls back to the unchanged boxed-storage read.

    + reassignment void: AssignStmt/CompoundAssign arm now voids _known_floatarr
      alongside _known_intarr (no stale proof after a re-let / write).

PART 2 — no-escape AUTO-FIRE (GATED opt-in -> default ON when proven safe):

  self/codegen.hexa:
    + _native_arr_noescape_names set + _native_arr_noescape_reset/add/
      _is_native_arr_noescape.
    + _native_arr_noescape_scan(body): per immutable `let xs = [int-lit…]` /
      `let xs = [f64-lit…]`, REUSES the F2 escape analyzer _stmt_escapes_name /
      _expr_escapes_name (the escape-stack analyzer) over EVERY sibling statement.
      The analyzer treats `xs[i]` element reads + `xs.len()` as LOCAL (non-escape)
      and EVERY other mention (call-arg, return, let-flow, store, aggregate
      literal, bare operand) as an ESCAPE. A binding with zero escapes is added to
      the proof set.
    + _native_arr_autofire_ok(name): returns true if the binding is proven
      non-escaping (AUTO-FIRE, no env flag) OR the legacy HEXA_NATIVE_ARR env flag
      is set (escape hatch / isolated-lowering test). Wired into all four
      construction + read sites (replacing the bare HEXA_NATIVE_ARR gate).
    + gen2_fn_decl: calls _native_arr_noescape_scan(node.body) (fn-scoped) next to
      the existing _stack_noescape_scan / _known_int_accum_scan pre-scans.

  Net effect: a read-only-loop monomorphic int/f64 array literal whose binding is
  statically proven non-escaping now lowers to native contiguous storage by
  DEFAULT (no flag). An escaping binding (returned / passed to an opaque fn /
  stored / put in an aggregate) keeps the boxed HexaArr representation — provable
  no-regression, decision is escape-driven not name-driven.
