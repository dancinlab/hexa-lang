WHICH N204 TECHNIQUE MOVED IT (rank by direct attribution)
  (a) 64x64 mma.sync large tiles  -- BACKFIRED. The bigger output tile multiplies the
       smem need (S 64x64 f32 = 16 KB vs round-4's 16x256 f32 = 16 KB ... actually same
       size for S, but the WHOLE Q+K+V+Vt+P+O resident set scales with BM=64).
  (b) TMA cp.async.bulk.tensor   -- NEUTRAL. Latency hidden by mbarriers; but only 1
       CTA/SM means no occupancy to absorb the wait. SAVED HBM transactions (5 TMA
       vs the round-4's 16-subtile-per-block global loads) but the win is canceled
       by occupancy loss.
  (c) double-buffer K/V          -- ZERO. (measured above)
  (d) SWIZZLE_128B               -- NOT APPLIED (v2 uses SWIZZLE_NONE to avoid the
       trans-fragment hazard discovered during the mma probe campaign).
  (e) V-transpose in smem        -- COST: extra 8 KB smem + 4096-element transpose
       per KV block. Necessary for the proven non-trans mma path (probe2/probe3
       empirically established ldmatrix-trans gives a block-wise transpose, not a
       full transpose, ruling out the trans path for P.V).

KEY DISCOVERY (genuine new knowledge, paper_significance candidate but ORANGE -> NO paper)
  The mma.sync.m16n8k16 trans/non-trans + ldmatrix combination was EMPIRICALLY
  established for the fused-attention case:
    probe2: non-trans ldmatrix B + mma.row.col => C = A . B^T  (err 1e-7, MATCH)
    probe3: P.V via V pre-transpose + non-trans mma => P.V    (err 6e-8, MATCH)
  This SUPERSEDES my initial design which used .trans for QK^T (matching the N204
  SGEMM precedent). The .trans modifier performs only an 8x8-block transpose
  within each 16x16 ldmatrix.x4 tile -- NOT a full 16x16 transpose -- making it
  unsuitable for the P.V GEMM's contraction-along-B-rows requirement.

CLOSED-NEGATIVE FINDING (terminal)
  "The N204 standalone-GEMM roofline toolkit (large 64x64 tile + double-buffer +
   TMA bulk loads) DOES NOT TRANSPLANT to fused flash-attention. Fused attention
   has a fundamentally different smem residency profile (Q + K + V + V^T + S + P + O
   simultaneously resident, all O(BM x BK) or O(BM x d)) that scales smem by
   BM^2 -- so the GEMM-optimal BM=64 forces 1 CTA/SM occupancy, regressing the
   wall from 3.47x (round-5, BM=16) to 5.3-6.9x (round-7, BM=64) despite a
   CORRECT and CORRECTLY-NUMERIC implementation of TMA + double-buffer + mma.sync."
  Binding constraint identified: smem-bound occupancy at 1 CTA/SM dominates.
  Path forward (NOT this round): a SHALLOWER tile (BM=32 or BM=16 with multi-warp)
  + REGISTER-RESIDENT O (avoid smem O tile) + selective TMA (Q only or interleaved)
  could restore occupancy WITHOUT abandoning TMA loads. This is the round-8 wedge.
