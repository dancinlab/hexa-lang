--- THESIS --------------------------------------------------------------

NOVEL axis (TECS-L axis F) is not only a discovery lane. It is also a
verify-infra growth driver. When a NOVEL round probes a new identity /
hypothesis, it routinely exposes a calc-fn gap (missing fn / fn ne spec).
The reflex of routing that gap through the g59 INBOX upstream pipe makes
every NOVEL round a lower-bound input to hexa-lang stdlib / compiler /
verify growth.

  NOVEL round -> calc-fn gap surfaced
              -> honest-tier verdict persisted
              -> g59 INBOX upstream entry
              -> hexa-lang patch (other session)
              -> next NOVEL round uses grown fn


--- CANONICAL 5-STEP PIPELINE -------------------------------------------

  (1) NOVEL round (verify try):
        hexa verify --expr | atlas atom | fence -- gate g5
  (2) honest tier record:
        write the exact verdict tier
        (red CLOSED-NEGATIVE / yellow CITATION / orange DEFERRED / white SPEC)
        persist verbatim to .verdicts/<slug>/<id>.txt
  (3) g59 INBOX upstream (reflex):
        prepend to INBOX.log.md a header + quant data + recommended actions
        cross-link verdict path + doc path
  (4) hexa-lang patch (other session):
        another agent reads the INBOX entry as anchor
        lands stdlib / compiler / verify fix; binary rebuild if needed
  (5) next NOVEL round (grown infra):
        new fn / new spec is available
        downstream milestones (other axes too) gain trust baseline
