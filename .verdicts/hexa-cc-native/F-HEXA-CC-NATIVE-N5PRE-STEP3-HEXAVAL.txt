value is a HexaVal at runtime and the runtime dispatches on its tag (the exact rationale the
existing unit/unknown tolerances already cite). Added `_is_hexaval(t)` and tolerated it as a
wildcard in four sites:
  1. _types_assignable: HexaVal on either side -> assignable (covers HX3004 return + HX3001 then/else/assign)
  2. if-condition gate: HexaVal cond -> not a bool-mismatch (HX3001)
  3. while-condition gate: same
  4. callee-must-be-fn: HexaVal callee -> callable, no HX2003
  5. (regression fix) dynamic HexaVal callee -> skip the HX3002/HX3003 arity/arg-type checks
     (arity unknown; runtime dispatches), return a dynamic (empty) result type.
