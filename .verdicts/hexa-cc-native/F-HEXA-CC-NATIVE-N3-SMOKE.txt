BUILDING BLOCK PRESENT
  self/codegen/runtime_arm64.hexa defines rt_exit / rt_exit_value (+ write,
  mmap) as raw ARM64 instruction-byte producers ("replaces runtime.c"). So
  the bytes for _hexa_exit exist natively; they are not yet packaged as a
  linkable MH_OBJECT exporting _hexa_exit / _hexa_set_args.

FINDING — N3 frontier is precisely TWO wiring tasks (not new capability)
  N3a. RUNTIME OBJECT: emit a native runtime .o (from runtime_arm64.hexa via
       compiler/emit/macho_arm64.hexa's serializer) that EXPORTS the runtime
       symbols a compiled program references — minimally _hexa_exit,
       _hexa_set_args; generally the _hexa_* set the native backend emits.
  N3b. LINK SWITCH: make --emit=exec --backend=native call hexa_ld.link()
       over { user.o, runtime.o } instead of forking system `ld`. The linker
       already merges multi-.o + resolves BRANCH26 (P1_LINKEXEC PASS) and
       does ad-hoc codesign (hexa_ld v1.2+); it needs the runtime.o as the
       second input + entry/_start wiring.
  Then a trivial program (exit 42) links with clang/ld/as fork = 0 and runs
  → that is the N3 closing falsifier (= a real P3_FULL_RUNEQ via aprime_cc,
  not `hexa run`).
