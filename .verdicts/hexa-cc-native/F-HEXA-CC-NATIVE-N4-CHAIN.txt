FALSIFIER (pre-registered)
  A runnable arm64 binary built from a .hexa source via the native chain
  (aprime_cc --emit=obj -> hexa_ld) must exit with the program's code while
  producing ZERO C intermediates (no .c, no ar-.a, no clang-compiled .o) and
  forking ZERO clang/as/ld. Any failing assert FALSIFIES the clang-free chain.

METHOD + VERIFICATION (real run, stdout verbatim)
  $ HEXA_LANG=<repo> hexa run \
      compiler/test/macho_p0_corpus/run_F_N4_CHAIN_ZERO_EXTERN.hexa
  F-N4-CHAIN-ZERO-EXTERN PASS — native user.o + runtime.o + hexa_ld
    -> Mach-O executable, exit 42 · .c=0 .a=0 native.o=2 · clang/as/ld fork 0
  (RUN_RC=0)

  The test asserts, in order:
   1. aprime_cc --emit=obj --backend=native trivial.hexa -> user.o:
      Mach-O magic cffaedfe; nm -u == {_hexa_exit, _hexa_set_args}.
   2. hexa run tool/emit_runtime_obj.hexa <w>/runtime.o (N3a):
      size > 0; nm -U exports _hexa_exit + _hexa_set_args; nm -u empty.
   3. hexa run tool/hexa_ld.hexa -o <w>/prog user.o runtime.o --lc-main _main:
      output is a Mach-O EXECUTE (magic + `file` "executable").
   4. <w>/prog -> exit 42.
   5. zero C-intermediate: find <w> -> .c=0, ar-.a=0, .o == exactly 2 (the two
      native objects; no clang-compiled artifact).
   6. fork-0: re-run step 1 with PATH pointing at an empty dir (clang/as/ld
      absent) — native obj-emit still produces a valid Mach-O .o, so it forks
      no C toolchain (the F-P3-ZERO-EXTERN technique). hexa_ld's fork-freedom is
      cited from F-P1-LINKEXEC (2 .o -> runnable, no clang/ld/as).

FINDING (Δ vs baseline)
  The N3a (runtime .o) + N3b (hexa_ld exec-link) win is now a single committed
  REGRESSION GUARD. Any future change that reintroduces a C intermediate, a
  clang/as/ld fork, or breaks the exit-42 native chain trips this falsifier.

  Note (corrects a prior false-halt): an earlier N4 attempt reported
  emit_runtime_obj.hexa wrote a 0-byte runtime.o because it read its output arg
  from args[2]. That was already fixed to args[1] in N3b (#2249, on origin/main)
  — `hexa run <script> <OUT>` puts OUT at argv()[1]. Re-verified on main: the
  emitter writes a 315-byte runtime.o to the given path (nm -U: _hexa_exit@0,
  _hexa_set_args@0x14), and the full chain reproduces exit 42 deterministically.

SCOPE / HONEST CAVEATS (g5/g63)
  - trivial/exit-only program · arm64-apple-darwin · committed build/aprime_cc
    (no rebuild). Larger programs reference more _hexa_* symbols (the N3a
    emitter extends); full-compiler self-host via hexa_ld is N5.
  - The test SKIPs (exit 0) when HEXA_LANG is unset or build/aprime_cc is absent
    (Linux nobaseline-gate runners), so the gate stays green where the native
    darwin toolchain is not present — the guard is darwin-arm64-scoped by design.
  - `hexa`/`aprime_cc` themselves are the only external tools in the chain
    (runners, not a C toolchain); the falsifier's "0 fork" is about clang/as/ld.
