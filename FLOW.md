# FLOW — 미니멀 작업흐름

배경: nexus/DRILL.md, hexa-bio/AXIS.md 두 세션이 같은 결함으로 무너짐
(순수 누적, 운영화 0건, 수사적 "고갈 선언"). FLOW는 그 결함을
역설계해 박은 최소 사이클.

```
풀기  →  자르기  →  잡기
divergence  prune  handle
```

## 세 동사

**풀기.** 무필터 생성. 시간/개수 박스로 종료. "고갈" 금지 —
박스 만료가 유일한 정지 조건.

**자르기.** delete ≥ add. 못 자르면 풀기로 회귀
(생산물이 솎을 만큼 충분하지 않았다는 뜻).
DRILL/AXIS가 한 번도 통과 못 한 관문.

**잡기.** 살아남은 항목에 1줄 measurement protocol.
못 잡으면 다시 자르기.

## 루프 종료 — 두 가지뿐

- **잡기 성공** → 도구 손잡이 생김. 끝.
- **리스트가 비어버림** → 시드가 약했음. 정직한 비-시작.
  발화로 돌아가 외부 자극 다시.

둘 다 정보 이득. 그 외의 "고갈/끝/마감" 선언은 모두 자르기 회피.

## 안티패턴 (3개)

- 풀기 2회 연속 = 자르기 회피
- `+`만 있는 md update = 자르기 실패
- 메타-브레인스토밍 = 풀기의 위장형, 1회 한도

## 그림

```
   ┌─→ 풀기 ─→ 자르기 ─→ 잡기 ─→ 도구
   │             │           │
   │             └─ 부족 ────┘
   └─ 약한 시드 ←─ 빔
```

세 동사·두 출구·세 금기.
더 줄이면 정체성 소실 (`자르기` 빼면 DRILL/AXIS로 회귀).
더 늘리면 회복하려던 결함이 다시 들어옴.
