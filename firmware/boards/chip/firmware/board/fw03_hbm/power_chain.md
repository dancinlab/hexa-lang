# HEXA-CHIP-FW-03 — Power chain detail

> Phase E iter 2 detailed power chain spec for the HBM thermal-controller
> board. Falsifier: F-CHIP-3. Builds on `schematic_paper.md §2` and
> `firmware/doc/board_v0_spec.md §3.3`. **Highest power density** of the
> three boards: HBM4 12-Hi stack drives ~35 W peak through a single
> Renesas RAA489204 multi-output PMU, with HBM4-spec sequencing.

## 1. Rail tree (current-budgeted)

```
  24 V DC barrel jack (or 4-pin Molex 12 V/5 V from ATX PSU)
        │  ≤ 2.0 A typ (48 W) / 2.5 A peak (60 W)
        │
        ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │ U1: Renesas RAA489204  Multi-output PMU                             │
  │   • V_in:  4.5–60 V                                                 │
  │   • 5 outputs, integrated sequencer with programmable order/delay   │
  │   • SMBus / I2C interface for fault status + DVFS slot              │
  │   • Output configuration (per HBM4 12-Hi stack):                    │
  │     ┌────────────┬─────────┬─────────┬───────────┬─────────────────┐│
  │     │ output     │ V_set   │ I_max   │ ripple    │ load (typ/peak) ││
  │     ├────────────┼─────────┼─────────┼───────────┼─────────────────┤│
  │     │ VOUT_A     │ 1.10 V  │ 12 A    │ < 5 mV    │ 4 A / 9 A       ││
  │     │ VOUT_B     │ 0.85 V  │ 4 A     │ < 4 mV    │ 1 A / 2.5 A     ││
  │     │ VOUT_C     │ 0.40 V  │ 8 A     │ < 3 mV    │ 2 A / 5.5 A     ││
  │     │ VOUT_D     │ 1.80 V  │ 2 A     │ < 8 mV    │ 0.2 A / 0.5 A   ││
  │     │ VOUT_E     │ 3.30 V  │ 1 A     │ < 15 mV   │ 0.15 A / 0.3 A  ││
  │     └────────────┴─────────┴─────────┴───────────┴─────────────────┘│
  │   • η:  87 % aggregate at 25 W total load                           │
  │   • Decoup (input): 220 µF + 100 µF + 22 µF + 100 nF                │
  └─────────────────────────────────────────────────────────────────────┘
```

Total board input load:
- Typ:   24 V × 1.05 A ≈ **25 W**  (HBM stack idle + PMU losses + host MCU active)
- Peak:  24 V × 1.85 A ≈ **44 W**  (HBM stress test full-load all 12 layers active)
- PSU sizing: 24 V / 2.5 A (60 W) brick → 1.4× headroom (worst on 3 boards).
- Aggregate HBM4 stack budget: **35 W peak** (matches `board_v0_spec §3.3`).

## 3. Power-on sequencer (RAA489204 internal)

HBM4 spec mandates rising-edge order so DRAM core (1.10 V) and LBD
logic (0.85 V) come up before any I/O is enabled — otherwise
parasitic currents through pad clamps damage cells.

```
   t (ms)  ┌─ VOUT_A (+1V10)  PG = 1
           │   ┌─ VOUT_B (+0V85) PG = 1
           │   │   ┌─ VOUT_C (+0V40) PG = 1
           │   │   │   ┌─ VOUT_D (+1V80) PG = 1
           │   │   │   │   ┌─ VOUT_E (+3V30) PG = 1
           │   │   │   │   │   ┌─ PG_ALL → HBM4 RESET_n released
   0───────┴───┴───┴───┴───┴───┴────────────────────────────────
           0   3   8   16  25  35

   Each slot: ramp time ≤ 2 ms, monotonic, settle within 90% × V_set.
   t_delay between PG_n and SLOT_n+1 enable: 5 ms (PMU programmable).
```

**CATTRIP override (paper-tier definition):**
TLV3702 thermal latch (driven by ADS1115 over-temp output) →
`TRIP_LATCH` net → RAA489204 `EN_GLOBAL` pin → all 5 rails to 0 V
**within 100 µs** of `T_max ≥ 105 °C` event. Hysteretic re-arm requires
host MCU `RESET_n` pulse + `T_max < 90 °C` gate.

## 4. Estimated rail noise / Z_target (paper-tier)

| rail    | est. ripple   | Z_target (DC–500 MHz) | margin to HBM4 spec        |
|:--------|:--------------|:----------------------|:----------------------------|
| +1V10   | < 5 mV (0.45%)| < 5 mΩ (HBM4 core)    | ~1.4× — VERY TIGHT          |
| +0V85   | < 4 mV (0.47%)| < 12 mΩ (LBD)         | ~2×                         |
| +0V40   | < 3 mV (0.75%)| < 8 mΩ  (DDR I/O)     | ~1.5×                       |
| +1V80   | < 8 mV (0.44%)| < 30 mΩ (peripheral)  | ~3×                         |
| +3V30   | < 15 mV (0.45%)| < 80 mΩ (MCU I/O)    | ~5×                         |

**Critical:** +1V10 HBM4 core. JEDEC HBM4 PI spec requires < 5 mΩ from
DC to 500 MHz. Bulk POSCAP × 4 (470 µF) covers DC–10 kHz; mid-band
covered by 22 µF MLCCs × 30 distributed under HBM stack interposer;
HF (10 MHz–500 MHz) covered by 100 nF + 10 nF pairs at every BGA ball
of HBM4. Buried + blind vias (per `schematic_paper §6`) reduce loop
inductance by ≈ 60 % vs through-hole.

## 5. Failure modes (FMEA-lite)

| failure                                  | detection                        | response                                   |
|:-----------------------------------------|:---------------------------------|:-------------------------------------------|
| 24 V brick over-current trip             | board off; no PMU log            | replug; LED on PSU                         |
| RAA489204 SMBus dead                     | host MCU can't query fault regs  | nRESET pulse on PMU; if persists → repair  |
| VOUT_A over-current (> 12 A)             | HBM4 core fold-back              | PMU asserts FAULT → MCU latches event      |
| VOUT_C UV during HBM4 init               | training pattern fails           | re-attempt init after PG_C re-asserts      |
| Sequence violation (1V8 before 1V1)      | PG_ALL never asserts             | latch; manual recovery via PMU re-program  |
| CATTRIP (T_max ≥ 105 °C)                 | TLV3702 latch                    | rails to 0 V < 100 µs; cool-down + re-arm  |
| Decoup short under HBM stack             | rail droop; fault flag           | thermal camera diagnosis (post-mortem)     |
| Si interposer crack (mechanical)         | DQ failures across multiple lanes| PMU shutdown; board write-off              |
| HBM4 PHY training stuck                  | RST_n cycled, training pattern bad| reset stack, re-init at slower DVFS slot  |

## 6. PI verification path (Phase E iter ≥ 3)

- [ ] HyperLynx PI on +1V10 plane: target ≤ 5 mΩ flat DC–500 MHz
- [ ] HyperLynx SI on HBM4 DQ (2048-bit) and DQS (32-bit) pairs
- [ ] Buried/blind via stub minimisation: < 0.5 mm un-used stub
- [ ] Layer-by-layer Si interposer alignment check (paper diff ≤ 5 µm)
- [ ] CATTRIP latency measurement: force T-rise step, scope rail collapse
- [ ] 24-h stress: 35 W full-load, T_J max < 105 °C continuous
- [ ] DVFS engagement: walk PMU slots, confirm rail set-points match HBM4 spec at each slot
- [ ] Power-cycle 100× → no infant mortality of MLCCs near HBM4 stack

(Deferred to G2/G5; full PI sweep needs HyperLynx PI + thermal chamber.)
