## 2. Workload Cost Model

### 2.1 RFC 072 P4 — Hexa-emit d=4096 24L on H100 80GB

**Spec recap (from GPU.md §10.1 + task brief):**
- d_model=4096, n_layer=24, seq_len=2048, batch=8
- 12 GB blocks d=2048+ on RTX 5070; H100 80GB has memory headroom (~5×) and HBM3 bandwidth.

**Sub-fires:**

|----------|---------------|------------:|------------:|-----:|
| (a) PyTorch eager d=4096 24L baseline ×3 variance runs | RunPod Community H100 SXM | 30 min (setup + 3 step1 runs) | $2.69 | **$1.35** |
| (b) Hexa-emit fire d=4096 24L ×2 measurements + variance | RunPod Community H100 SXM | 1 hr (compile + run + recompile + 2 runs) | $2.69 | **$2.69** |
| (c) Buffer (network egress, ROCm-style CI setup retry, 1 reboot) | same | 30 min | $2.69 | **$1.35** |
| **Subtotal** | | **2 hr** | | **$5.39** |

Add ×1.5 safety margin (vast.ai/RunPod cold-start, container pull, `pip install torch` 8GB, hexa toolchain bootstrap): **~$8** realistic, **$12** worst-case.

If we move to Vast.ai verified-host ($1.87/hr): subtotal drops to ~$3.74, realistic ~$5.50. **But:** Vast.ai per-host network egress and disk are flaky; first run often spent debugging the host, not the model. RunPod is the predictable choice for a closure-grade measurement.

**Estimate range: $8 – $12 (target: <$15).**

## 3. Recommended Sequencing

**Phase A — MI300X first (cheaper, lower-risk):**
- $4-8 spend
- Validates Metal→HIP/ROCm codegen path (RFC 075 P3 already landed MSL emitter; P4 only needs the HIP backend re-emit + one fire).
- If this fails, the bug is in our codegen, not in workload sizing — cheap diagnostic.

**Phase B — H100 d=4096 second (higher-risk, depends on RFC 072 codegen being clean):**
- $8-12 spend
- Run only after MI300X fire confirms multi-vendor RFC 075 backend works on at least one non-NVIDIA target.
- Pair with RFC 072 §8 "honest caveats" PTX-diff oracle BEFORE provisioning — if `ptx-diff` shows 5× ILP gap vs nvcc, fix that first before paying for the d=4096 fire.

**Hold/defer:**
- Crusoe MI300X spot ($0.95/hr) — defer until we have a checkpoint protocol. Saves ~50% but preemption mid-fire wastes wall + measurement.
- Vast.ai H100 — defer until RunPod fire produces baseline; use Vast.ai only if we need a 3rd-vendor cross-check at the lowest dollar floor.
