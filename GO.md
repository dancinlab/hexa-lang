# GO — hexa-lang `hexa run/build` Go-ergonomic 정리 도메인

> 한 줄: hexa-lang 의 사용자-대면 CLI (`hexa run`/`hexa build`) 를 Go (`go run`/`go build`) 수준의 무신경(no-friction) 사용감으로 정리.

@goal: hexa-lang CLI 가 캐시·디스크·재컴파일·toolchain 발견 전반에서 사용자가 신경 쓸 게 0 인 상태가 된다 (Go `go run`/`go build` 수준).

- [x] M2 — `tool/build_precompile.hexa` 파이프라인 + `self/main.hexa` cmd_run lookup precompile dir 우선 probe + `tests/m_precompile_hit_test.hexa` e2e. cold-cache fork-storm 의 *내부* axis (release 시점 precompile → 사용자 첫 호출 즉시 cache HIT).

(R3 = autospawn wiring · binary length-prefix wire · N=100 latency · crash-respawn.)
- [x] M14 — hexa daemon R4(c) atlas SSOT hash flush (RFC 093 Phase 4 · F-DAEMON-4). daemon serve-loop 가 COMPILE 처리 직전 (2 s throttle) atlas SSOT 파일 (`compiler/atlas/embedded.gen.hexa`, `$HEXA_LANG > install_dir > ./` 해석) 의 content hash (`sha256_file`) 를 직전 값과 비교 → 변경 시 in-memory cache (`cache_keys`/`cache_bins`) 전체 flush + 해당 `~/.hexa-cache` disk-mirror 바이너리 unlink → 다음 COMPILE 이 새 atlas 로 re-build (stale HIT 방지). **Option B (daemon-only invalidation layer)** — `cmd_run`/precompile 키 불변 (`sha256(source)[0:16]+"_"+version_str()`) → M7 drift-guard · M2 precompile 무영향. (R4 잔여 = (a) binary u32-LE wire · (b) in-mem lexer/parse/lower · (d) multi-uid reject F-DAEMON-5 · (e) prod ship.)
- [x] M12 — hexa daemon R3 (RFC 093 Phase 3). `hexa run` **autospawn wiring** — `HEXA_DAEMON_AUTOSPAWN=1` (opt-in · default 0 → 행동 변화 0건) 시 socket probe → spawn-if-missing (`daemon start-bg`) → `COMPILE <src>` → 받은 binary path exec. 3-tier fallback **precompile (M2) → daemon (R3) → fork-mode** 정립 (`cmd_run_user_direct`). daemon 실패(socket missing+spawn fail · ERR · dead daemon)는 silent fork fallback (graceful degrade). `HEXA_DAEMON_SOCKET` env override 추가. `tests/m_daemon_r3_test.hexa` 3 falsifier: F-DAEMON-R3-1 (autospawn) · F-DAEMON-R3-2 (latency) · F-DAEMON-R3-3 (crash-respawn). **측정 (mini arm64 fresh build): N=100 warm-daemon 9,395 ms vs fork-mode 174,933 ms = 18.6× speedup** (F-DAEMON-2 RFC gate ≤ 1.5× far exceeded). R1/R2 회귀 무 (4/4 + 3/3 PASS). `net_read_raw` 는 occam g0 로 skip (binary path 는 NUL 없어 newline-text round-trip OK) → R4 로 연기. (R4 = binary length-prefix wire · in-mem lexer/parse/lower · atlas hash flush · multi-uid reject · prod ship.)
- [x] M13 — hexa daemon R4 codegen (RFC 093 § R4 carve-out). 2 신규 codegen builtin: **`net_read_raw(fd, len)`** = 정확히 len byte NUL-preserving read ([int] 반환 · net_read_n 의 exactly-n 루프 + net_read_bytes 의 NUL-safe array 결합 — net_read/net_read_n 은 strlen-truncate, net_read_bytes 는 single-recv 64 KiB cap) · **`os_getuid()`** = getuid(2) wrapper (`getuid` 가 reserved-name 이라 `u_getuid` 로 mangle → dlsym 충돌 → `term_getppid` 처럼 비충돌 이름 `os_getuid` 채택). 그 위에 **F-DAEMON-5 multi-uid 강화**: socket path uid-scoped (`/tmp/hexa-daemon-<uid>.sock`, `$USER` → 실 numeric uid) + root-refuse (`os_getuid()==0 → exit 1`). codegen 레시피 엄수 — `hexa cc --regen` → promote → DIRECT transpile 검증 → **fixpoint PASS (gen2≡gen3 byte-identical hexa_cc.c)** → 회귀 무 (new==base clang-err). `tests/m_daemon_r4_codegen_test.hexa` 3 falsifier ALL PASS: R4-1 (loopback NUL round-trip 7 bytes 정확 복원) · R4-2 (os_getuid==`id -u`=501) · R4-3 (uid-scoped socket + non-root). **binary length-prefix wire 전환은 R5 deferred** — occam g0: 현 daemon payload (path·status) 는 NUL 없어 newline-text 충분; net_read_raw builtin 만 선반영. (R5 = binary wire 실전환 + atlas hash flush + SO_PEERCRED per-conn uid 확인 + prod ship.)
